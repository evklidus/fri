# 5-минутный сценарий записи демо для инвестора

> Инструмент: Loom / QuickTime / OBS. Запиши экран + микрофон. Все шаги ниже — **уже работают** в текущем окружении (`make docker-up` → http://localhost:8080).

## До записи

1. Запусти стек:
   ```bash
   make docker-up
   # дождись 'FRI API listening on :8080'
   ```
2. Открой Chrome, размер окна — 1440×900 (или больше).
3. Открой http://localhost:8080.
4. Открой второй tab с терминалом сбоку — для команд `make sync-all`.
5. Включи запись.

---

## Сценарий (5 минут)

### 0:00–0:30 — Контекст

> «FRI — Football Reputation Index. Раньше это был статичный HTML с захардкоженными цифрами. Сейчас это живая система, которая автоматически собирает данные из пяти источников и пересчитывает рейтинг футболистов в реальном времени.»

**Покажи**: главная страница. Hero-карточка с **H. Kane** и его FRI **78.4**.

> «Видите карточку наверху — это игрок с самым высоким текущим FRI. Обновляется после каждого sync.»

---

### 0:30–1:30 — Лидерборд и фильтры

**Скролл** до Rankings.

> «Здесь полный лидерборд — 22 игрока. Топ-5: Kane, Yamal, Mbappé, Haaland, Valverde. Это соответствует реальной картине сезона 2025/26 — Kane лидирует Бундеслигу, Yamal — открытие в Барселоне.»

**Кликни** фильтр «La Liga».

> «Фильтр по лиге — выбираем La Liga. Видим 12 игроков. Yamal, Mbappé, Bellingham, Vinicius, Pedri — Real Madrid и Барса.»

**Сбрось** на «All leagues», **кликни** «Forward».

> «Фильтр по позиции — нападающие. Девять игроков. Kane на первом месте.»

---

### 1:30–2:30 — Карточка игрока

**Кликни** на строку **M. Olise**.

> «При клике открывается детализация. Видим Performance 78.3, Social 27.8, Fan Poll 83.7, Media 71.4, Character 91 — пять компонентов FRI с весами 35/20/20/15/10.»

**Подожди 1 сек**, пока загрузится график.

> «Внизу — график истории FRI. Это реальные точки из БД, обновляются после каждого sync. Видно, что у Olise FRI колебался от 75 до 69.5 за месяц.»

**Скролл** в модалке вниз.

> «Здесь — лента новостей про Olise за последние 30 дней. Берётся из MediaStack, тональность анализируется VADER. Видим: "Crystal Palace to Europe's creator-in-chief" — позитивная статья, +1.2 к Media. Каждая строка влияет на FRI.»

**Закрой** модалку.

---

### 2:30–3:30 — Голосование и антиабьюз

**Скролл** до секции Fan Polls.

> «Фанаты могут голосовать. 20% веса FRI приходит от голосов. Голосую за Olise — пять звёзд, hype 9, GOAT-tier, role-model behaviour.»

**Заполни** форму, **нажми** «Submit». Появится «Vote Counted».

> «Голос учтён. Backend пересчитал Fan Score, FRI обновился, страница перерисовалась автоматически.»

**Открой** модалку Olise снова — покажи новый Fan Score (>83.7).

**Закрой** модалку, **снова** проголосуй за Olise.

> «А теперь попробую проголосовать ещё раз — за того же игрока с того же IP.»

Появится **error** «vote rate limit: already voted for this player in the last 24h».

> «Антиабьюз: один голос на игрока с одного IP в сутки. IP хешируется через SHA-256, raw IP в БД не хранится.»

---

### 3:30–4:30 — Sync и реакция на события

**Переключись** в терминал.

```bash
make sync-all
```

> «Запускаю полный sync — Performance, Social, Media, Character. Это эмулирует то, что происходит автоматически каждые 6–24 часа в фоне через scheduler.»

**Ждёшь ~60 секунд**, пока выполнится. Покажи JSON-output:

> «Видим: `performance: 22 players, character: 84 articles scanned → 3 events → 2 players moved`. Это значит — нашли свежие новости, обнаружили события (например, скандал или fair play award), пересчитали Character у двух игроков, FRI поехал.»

**Вернись** на страницу, **обнови** браузер.

> «Страница перезагрузилась — видим новые цифры в лидерборде.»

---

### 4:30–5:00 — Закрытие

> «Что мы видели:
> - 22 игрока, 5 компонентов, все на реальных данных из API-Football, MediaStack, YouTube и news-sentiment.
> - Голос фаната моментально влияет на FRI.
> - Антиабьюз голосования.
> - История FRI хранится — видно динамику.
> - Скандал в новостях → Character падает → FRI падает. Награда → FRI растёт.
>
> Дальше — расширение до 200+ игроков, Instagram/Twitter API, production deploy.»

**Стоп** запись.

## Команды для второго tab'а (копи-паст)

```bash
# Текущее состояние
curl -s http://localhost:8080/api/sync/updates | jq '.data[:3]'

# Запуск всех sync
make sync-all

# Только media (быстрее)
make sync-media

# Запуск character (показать что система ловит события)
make sync-character

# Health check
curl -s http://localhost:8080/api/health
```

## Если что-то пошло не так

| Проблема | Решение |
|---|---|
| Страница белая | `make docker-logs` → проверить что app started |
| Vote 500 | `make docker-logs` → проверить что миграция 002+ применилась |
| Sync долгий >2 мин | проверь `MEDIASTACK_API_KEY` и `API_FOOTBALL_KEY` в `.env` |
| Голос не даёт ошибку 429 | первый sync создал маппинг — попробуй второй раз через 1 сек |

## Итог

Запись 5 минут даст инвестору **полное представление** о проекте. Ключевые поинты:

1. **Это работает** — не моки, реальные API, реальные новости.
2. **Это живёт** — голос → instant. Sync → обновление в реальном времени.
3. **Это безопасно** — антиабьюз, IP hashed, sanity-check от false matching.
4. **Это масштабируется** — добавление игрока = одна строка в seed, остальное автоматически подтянется.
